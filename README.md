# boss-s-wiki (P-Reinforce Knowledge Engine)

**boss-s-wiki**는 강화학습(RL) 기반의 지식 자동화 에이전트인 **P-Reinforce Architect (The Autonomous Gardener)**에 의해 관리되는 영속적 위키(Persistent Wiki) 저장소입니다. 
사용자가 파편화된 정보를 던져두면, 에이전트가 의미론적 분류를 수행하고 지식 간의 상호 연결망(Knowledge Graph)을 구축하여 구조화된 하나의 거대한 '외부 뇌'를 만들어냅니다.

## 🚀 핵심 기능 (Core Features)
1. **자동 지식화**: 사용자의 가공되지 않은 원시 데이터를 실시간으로 모니터링하여 정제된 위키 포맷으로 자동 변환합니다.
2. **동적 폴더 트리 설계**: 고정된 폴더 구조에 얽매이지 않고, 새로운 지식이 들어오면 에이전트가 스스로 상위 개념을 도출해 카테고리를 확장합니다. 특정 폴더에 문서가 많아지면 하위 카테고리로 세분화(Refactoring)를 진행합니다.
3. **지식 융합 (Graph Connectivity)**: 관련 있는 개념, 원시 데이터, 정책 문서 등을 [[쌍방향 링크]]로 엮어 유기적인 연결망을 형성합니다.
4. **버전 관리 자동화 (Git Protocol)**: 지식이 추가되거나 구조가 개편될 때마다 에이전트가 내부적으로 GitHub에 커밋하여 지식의 타임라인과 진화 과정을 투명하게 보존합니다.

## 📂 폴더 구조 (Directory Structure)

```text
root/
├── 00_Raw/                 # [불변] 사용자가 입력한 가공되지 않은 모든 원시 데이터 (Source of Truth)
├── 10_Wiki/                # [자동 구조화] 에이전트가 RL 정책에 따라 관리하는 지식 층
│   ├── 🛠️ Projects/        # 목표 중심 (진행 중인 프로젝트 요약)
│   ├── 💡 Topics/          # 개념 중심 (철학, 코딩 등 스스로 생성한 개념 분류)
│   ├── ⚖️ Decisions/       # 의사결정 중심 (왜 이렇게 판단했는가에 대한 기록)
│   └── 🚀 Skills/          # 실행 중심 (프롬프트, 자동화 워크플로우 패턴)
├── 20_Meta/                # [시스템] 지식 엔진의 두뇌 데이터
│   ├── Graph.json          # 지식 간 연결 관계 데이터 (시각화 목적)
│   ├── Policy.md           # 사용자 피드백이 반영된 분류 정책 문서 (RL Weights)
│   └── Index.md            # 위키 전체 구조의 입구 (Table of Contents)
└── P-Reinforce_Skill.md    # 에이전트의 핵심 동작 로직이 담긴 시스템 지침서
```

## 🧠 에이전트 피드백 & 강화학습 가이드 (How to Train)

이 위키 시스템의 분류 엔진은 사용자의 피드백에 의해 최적화됩니다. (보상 함수 극대화)

- **칭찬하기**: "이 폴더 분류 완벽해" 👉 에이전트는 해당 주제의 유사도 가중치를 높이고 분류에 대한 확신도(Confidence Score)를 강화합니다.
- **수정하기**: "이 문서는 '코딩'이 아니라 '비즈니스'로 옮겨줘" 👉 에이전트는 두 주제 사이의 경계선(Boundary Shift)을 재설정하여 다음 분류 시 오차를 줄입니다.
- **방치하기**: 에이전트가 만들어낸 구조를 그대로 사용한다면, 이는 암묵적인 보상으로 간주되어 현재의 분류 정책이 유지(고착화)됩니다.
